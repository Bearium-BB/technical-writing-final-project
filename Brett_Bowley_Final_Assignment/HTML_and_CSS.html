Getting a quick start with the basics of HTML and CSS

Hello there! If you came to learn HTML and CSS you came to the right place. This is a quick guide to the basics of HTML and CSS so you can make your coding dreams come true.

What is HTML:

You may be asking what HTML is. It stands for HyperText Markup Language but it is commonly referred to as just HTML. Now let's get back to the Question for today. What is HTML? Web browsers can't understand English so you need the language, HTML. Humans can understand it and so can computers and that's how we designed web browsers on the base level.

Some basic things you need to know about HTML:

The primary thing you will use in HTML is a tag. You must be wondering what a tag is, aren't you? To answer that question for you, tags let the programmer add stuff to the web pages like images, text, and videos. Those are some things you can add stuff with HTML but there are many more. Here's a list of some basic tags.

  • H1
  • H2
  • H3
  • div
  • div
  • hr
  • Iframe
  • main

Now you know what a tag is, let's talk about elements. Simply put, elements are pretty much the same as tags. Without going into specific details they are mostly the same.

Now you know what an element is, let's talk about attributes. Attributes let you modify an element. There are a bunch of attributes in HTML but sadly we can't go over all of them so here's a list of some basic attributes.

  • Accept
  • Class
  • id
  • Form
  • height
  • width

What is an inline element and block element:

Pretty much would be an inline element if it only needs to take up a space it needs on the screen width, but on the other hand block elements take up the whole screen width.

The basics of CSS:

There's a term in CSS called a rule. You must be asking what is a rule in CSS? You have a bunch of modifiers that affect a single element, an example would be how you would type it out.

Now let's talk about selectors. What is a selector? It is pretty much what tag you want to edit, that's pretty much it. There is one more important rule, value. It's pretty self-explanatory. It's for sending values to the modifiers you add to it.
